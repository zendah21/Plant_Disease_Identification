# Plant Disease Identification

A Flask-based REST API that classifies plant leaf diseases from images using machine learning. The system uses a two-stage k-NN pipeline with Local Binary Pattern (LBP) features to identify diseases across 12 crop types, then returns symptom descriptions and treatment recommendations from a MySQL database.

## How It Works

1. Client sends a base64-encoded leaf image to the API
2. The image is converted to grayscale, resized to 128×128, and LBP features are extracted
3. A binary k-NN classifier determines if the leaf is healthy or diseased
4. If diseased, a multi-class k-NN classifier identifies the specific plant and disease
5. The API queries MySQL for symptom and treatment information and returns the result

## Tech Stack

- **API:** Flask
- **ML:** scikit-learn (k-NN, StandardScaler, GaussianRandomProjection)
- **Image Processing:** OpenCV
- **Database:** MySQL
- **Model Serialization:** joblib

## Database Setup

1. Start a MySQL server.
2. Run the SQL script to create the database and tables:

```bash
mysql -u root -p < "Plant Disease Treatments DB.db"
```

3. Update the database credentials in [server.py](server.py):

```python
db_config = {
    'host': 'localhost',
    'user': 'your_user',
    'password': 'your_password',
    'database': 'PlantDiseaseDB'
}
```

## Model Files

The server requires three pre-trained files in the project root:

| File | Description |
|------|-------------|
| `knn_model_binary.joblib` | Binary classifier (healthy vs. diseased) |
| `knn_model_multiclass.joblib` | Multi-class classifier (specific disease) |
| `scaler.pkl` | StandardScaler used during training |

These files are not included in the repository and must be generated by running the training script against the dataset.

## Running the Server

```bash
python server.py
```

## API Reference

### `GET /`

Health check.

**Response:**
```json
{"result": "Welcome to the server"}
```

---

### `POST /get_disease_info`

Diagnose a plant leaf image.

**Request body:**
```json
{
  "image": "<base64_encoded_image>"
}
```

**Response — healthy leaf:**
```json
{"result": "Healthy"}
```

**Response — diseased leaf:**
```json
{
  "result": {
    "disease_name": "Early Blight",
    "plant_name": "Tomato",
    "symptoms": "Dark spots with concentric rings appearing on older/lower leaves first...",
    "treatment": "Remove infected leaves, apply copper-based fungicide..."
  }
}
```

**Response — error:**
```json
{"error": "<error_message>"}
```

## ML Pipeline Details

### Feature Extraction (LBP)

- Input image → grayscale → resized to 128×128
- LBP computed over each pixel using 8 neighbors
- Flattened into a 16,384-dimensional feature vector

### Stage 1 — Binary Classifier

| Property | Value |
|----------|-------|
| Algorithm | k-NN (k=3, distance weighting, euclidean) |
| Dimensionality reduction | GaussianRandomProjection → 2000 components |
| Normalization | StandardScaler |
| Output | 0 = Healthy, 1 = Diseased |

### Stage 2 — Multi-class Classifier

| Property | Value |
|----------|-------|
| Algorithm | k-NN |
| Dimensionality reduction | GaussianRandomProjection → 1000 components |
| Normalization | StandardScaler |
| Output | `"PlantName___DiseaseName"` |
